Что такое Spring Framework?

Определение

Spring Framework — это комплексный фреймворк для разработки Java-приложений, который предоставляет инфраструктурную поддержку для создания Java-приложений. Spring берет на себя инфраструктурные задачи, позволяя разработчикам сосредоточиться на бизнес-логике.

Основные принципы

  • Inversion of Control (IoC) - Инверсия управления
  • Dependency Injection (DI) - Внедрение зависимостей
  • Aspect-Oriented Programming (AOP) - Аспектно-ориентированное программирование
  • Plain Old Java Object (POJO) - Поддержка обычных Java объектов

IoC контейнер

Что такое IoC?

Inversion of Control (Инверсия управления) — это принцип, при котором контроль над созданием и управлением объектами передается от самих объектов внешнему контейнеру.

Пример без IoC:

public class UserService {
    private UserRepository userRepository;
    
    public UserService() {
        // Жесткая связь - создаем зависимость сами
        this.userRepository = new UserRepositoryImpl();
    }
}

Пример с IoC:

@Service
public class UserService {
    private final UserRepository userRepository;
    
    // Зависимость внедряется контейнером
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

Dependency Injection

Типы внедрения зависимостей

1. Constructor Injection (рекомендуется)

@Service
public class OrderService {
    private final PaymentService paymentService;
    private final EmailService emailService;
    
    public OrderService(PaymentService paymentService, 
                       EmailService emailService) {
        this.paymentService = paymentService;
        this.emailService = emailService;
    }
}

2. Setter Injection

@Service
public class OrderService {
    private PaymentService paymentService;
    
    @Autowired
    public void setPaymentService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}

3. Field Injection (не рекомендуется)

@Service
public class OrderService {
    @Autowired
    private PaymentService paymentService;
}

Spring Beans

Что такое Spring Bean?

Spring Bean — это объект, который создается, конфигурируется и управляется Spring IoC контейнером.

Способы создания Bean

1. Аннотации

@Component
public class UserService {
    // ...
}

@Service
public class PaymentService {
    // ...
}

@Repository
public class UserRepository {
    // ...
}

@Controller
public class UserController {
    // ...
}

2. Java Configuration

@Configuration
public class AppConfig {
    
    @Bean
    public UserService userService() {
        return new UserService();
    }
    
    @Bean
    public DataSource dataSource() {
        HikariDataSource dataSource = new HikariDataSource();
        dataSource.setJdbcUrl("jdbc:mysql://localhost/mydb");
        return dataSource;
    }
}

Конфигурация

@Configuration

Аннотация для обозначения класса как источника определений bean'ов.

@ComponentScan

@Configuration
@ComponentScan(basePackages = "com.example.service")
public class AppConfig {
    // Spring будет сканировать пакет com.example.service
}

@Import

@Configuration
@Import({DatabaseConfig.class, WebConfig.class})
public class AppConfig {
    // Импорт других конфигурационных классов
}

Основные аннотации

@Autowired

Автоматическое внедрение зависимостей по типу.

@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;
}

@Qualifier

Указание конкретного bean'а при наличии нескольких кандидатов.

@Service
public class NotificationService {
    @Autowired
    @Qualifier("emailSender")
    private MessageSender messageSender;
}

@Primary

Указание приоритетного bean'а.

@Component
@Primary
public class EmailSender implements MessageSender {
    // Этот bean будет выбран по умолчанию
}

@Value

Внедрение значений из properties файлов.

@Component
public class DatabaseConfig {
    @Value("${database.url}")
    private String databaseUrl;
    
    @Value("${database.timeout:30}")
    private int timeout; // значение по умолчанию 30
}

Области видимости Bean'ов

Singleton (по умолчанию)

@Component
@Scope("singleton") // можно не указывать
public class UserService {
    // Один экземпляр на весь контекст
}

Prototype

@Component
@Scope("prototype")
public class ShoppingCart {
    // Новый экземпляр при каждом запросе
}

Web-scopes

@Component
@Scope("request")
public class RequestScopedBean {
    // Один экземпляр на HTTP запрос
}

@Component
@Scope("session")
public class SessionScopedBean {
    // Один экземпляр на HTTP сессию
}

Жизненный цикл Bean'а

Методы инициализации и уничтожения

@Component
public class DatabaseConnection {
    
    @PostConstruct
    public void init() {
        System.out.println("Инициализация соединения с БД");
    }
    
    @PreDestroy
    public void cleanup() {
        System.out.println("Закрытие соединения с БД");
    }
}

Интерфейсы жизненного цикла

@Component
public class MyBean implements InitializingBean, DisposableBean {
    
    @Override
    public void afterPropertiesSet() throws Exception {
        // Вызывается после установки всех свойств
    }
    
    @Override
    public void destroy() throws Exception {
        // Вызывается перед уничтожением bean'а
    }
}
